# Screenshot Guide

Add redacted product screenshots to this folder before sharing the repository with employers.

Recommended files:

1. `dashboard-task-console.png`
   Show the dashboard with live task progress, WebSocket updates, and manual review entry points.
2. `manual-review-card.png`
   Show the borderline-decision workflow and the generated material preview.
3. `resume-optimizer.png`
   Show target profile selection plus `add / modify / delete` edit instructions.
4. `resume-library.png`
   Show saved resume variants and default resume management.
5. `market-intelligence.png`
   Show the market analysis page with trend or skill insights.

Redaction rules:

- Remove personal email addresses, phone numbers, resume content, and API keys.
- Blur or crop real company names if they come from live scraped data.
- Do not include browser profiles, cookie data, or local file paths.
- Prefer one desktop screenshot and one mobile-width screenshot if the page is responsive.

Suggested capture flow:

1. Start the backend and frontend locally.
2. Log in with a demo account.
3. Navigate to each page and prepare stable example state.
4. Capture screenshots with consistent window size.
5. Save optimized PNGs into this folder and link them from `README.md`.
